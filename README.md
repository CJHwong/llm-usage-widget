# Ollama Gauge

A macOS menu-bar app that scrapes Ollama and ChatGPT Codex usage and shows it in a real Notification Center / desktop widget. It reuses your browser's existing login session, so there's nothing extra to authenticate.

The app is split in two:

- **Host app** (`Ollama Gauge.app`): unsandboxed menu-bar accessory. It scrapes every 5 minutes (Playwright/curl + browser cookies), writes `usage.json` into a shared App Group container, and tells the widget to reload.
- **Widget extension** (`OllamaGaugeWidget.appex`): sandboxed WidgetKit extension. It reads that same App Group container and renders the widget. The host is unsandboxed and the widget is sandboxed, but both can reach the group container, so there's one copy of the data and no mirroring.

## Tutorial: first run

Requirements:

- macOS 15+, Xcode 16+
- An Apple Development signing identity added under Xcode > Settings > Accounts (automatic signing registers the App Group; no portal visit needed)
- `xcodegen`: `brew install xcodegen`
- Zen or Firefox, signed into [ollama.com](https://ollama.com)

Build and install:

```bash
./build.sh
```

Generates the Xcode project, builds a signed `.app` with the embedded widget into `/Applications`, and registers a LaunchAgent that runs it at login. Then open Notification Center (or desktop edit mode) and add the **Ollama Gauge** widget.

## How-to guides

### Run it at login

`./build.sh` already installs the LaunchAgent. Logs from the scraper go through the unified logging system (`log show --predicate 'subsystem == "com.llmwidget"'`); LaunchAgent stdout/stderr go to `/tmp/llm-widget.log`.

Stop with `launchctl bootout gui/$(id -u)/com.llmwidget`. Restart with `launchctl kickstart -k gui/$(id -u)/com.llmwidget`.

Only one instance runs at a time (guarded by an `instance.lock` in the App Group container), so launching the app by hand while the LaunchAgent owns it is a no-op.

### Enable ChatGPT Codex usage

Needs `playwright-cli` and the Playwright Firefox bundle:

```bash
playwright-cli install-browser firefox
touch ~/Library/Group\ Containers/3GXP3XQ69M.com.hoss.ollama-gauge/chatgpt_enabled
```

Disable by removing that `chatgpt_enabled` file. Both the host (which scrapes) and the widget (which shows the section) read the same flag, so the next scrape picks up the change.

### If the widget does not appear in the gallery

The system (`chronod`) silently rejects widget extensions that are not properly sandboxed/entitled. If the widget is missing:

```bash
pluginkit -m -i com.hoss.ollama-gauge.widget     # should list it
pluginkit -a "/Applications/Ollama Gauge.app/Contents/PlugIns/OllamaGaugeWidget.appex"
log show --last 5m --predicate 'subsystem == "com.apple.chronod"' | grep -i ollama
```

Re-run `./build.sh` (which calls `pluginkit -a`) and re-login if it still won't register.

## Reference

### Project layout

```
LLMUsageWidget/
  project.yml            xcodegen spec: app + widget extension targets
  App/                   host app (unsandboxed): Scraper, AppPaths, BrowserAdapter, App entry
  Widget/                widget extension (sandboxed): TimelineProvider + widget views
  Shared/                Models + UsageDataStore (App Group bridge) + shared SwiftUI views
```

The `.xcodeproj` and `build/` output are generated by xcodegen/xcodebuild and gitignored.

### Browser support

Firefox-family only. The host reads cookies from a Firefox `cookies.sqlite`, so support is limited to:

- Zen
- Firefox

Chrome, Safari, Arc, and Edge are **not** supported. You must be signed into the sites in one of the supported browsers.

### Data directory: the App Group container

Everything lives in `~/Library/Group Containers/3GXP3XQ69M.com.hoss.ollama-gauge/`, shared by the host and the sandboxed widget:

- `usage.json`: last scraped payload
- `chatgpt_enabled`: presence enables ChatGPT scraping (read by both host and widget)
- `instance.lock`: single-instance flock

### Preflight checks

The scraper logs any of these instead of producing data:

- no supported Zen/Firefox profile found
- missing `sqlite3` or `curl`
- ChatGPT enabled without `playwright-cli`
- ChatGPT enabled without the Playwright Firefox bundle

## Explanation: how it works

**Ollama:** reads session cookies from the browser profile and fetches `ollama.com/settings` with `curl`, then parses the usage meters out of the HTML.

**ChatGPT:** launches a Playwright Firefox session that reuses the browser's cookies, opens the Codex analytics page, and polls until it renders before parsing.

Both run locally in the host app. The widget extension only reads the mirrored cache; it never scrapes. The only network traffic is the requests to ollama.com and chatgpt.com that your browser would make anyway.
